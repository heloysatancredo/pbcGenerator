About this Project

Coding and Learning!

HTML, CSS and JavaScript

This is the Final Project for the courses of HTML, CSS and Javascript from Term 1 for my post-degree program - Web & Mobile Design and Development, Langara College. I spent 2 weeks building this website and tried to apply much of the knowledge I gained in the last 3 months. Before that, I was not able to write 1 line in JavaScript. I still have a long way to go, but I am proud of this work. Special thanks to the instructors Denis and Reza for the knowledge passed on to me.

The requirements included:

HTML

  • One (1) home page and more six (6) pages

CSS

  • One (1) external style sheet (CSS) which contains a minimum of six (6) styles

JavaScript

  • Appropriately named: IDs, Variables and Functions
  • EventListeners, Functions, Arrays and Objects
  • Conditionals, Decision (if or switch) and Looping
  • Any three (3) of the following coding challenges:
       - Radio Buttons
       - Check Box(es)
       - Colour Picker/Setter
       - Date Picker/Setter
       - Input Box
  • All code must pull the user’s selection from the user interface, process the selection accordingly, and update the user interface with an appropriate acknowledgement or response.

General - Each page must:

  • Contain appropriate text and images that provide information and value to the target audience
  • Contain appropriate page titles
  • Be designed with responsive layouts that display well on both desktop and mobile browsers
  • Have a unified design across all pages (ie: headers/footers are the same, etc.)
  • The usage of frameworks or libraries is not allowed.

**Applying JavaScript**

This project contains 2 external JS files:

1) main.js: Included in all pages. It contains minor events and functions (mostly related to show/hide the side menu).

2) generator.js: Included in generator.html page. It provides dynamic update of the business card as the user fills in the fields and chooses appearance selections.This file contains:
  • Multidimensional Array of available images to the user to choose when building his business card
  • Object to keep data inputted by the user when building the business card
  • Function to update the object values
  • Function with 4 conditions and loopings to define which image in which color should be displayed according to user’s selection
  • EventListener to update the business card data according to the information provided by the user on 5 Input Boxes (Name, Professional Occupancy, Email, LinkedIn/ Social Media, Phone Number
  • EventListener to update the business card appearance according to user’s choices of background and font colors (2 Color Pickers*)
  • EventListeners calling function to update the business card image according to user’s selection of type and color (Radio buttons)
  • EventListener reloading the page if the user chooses to build a new business card
  • EventListeners to open and close the business card Settings (when displaying in mobile size
  • EventListener to validate: if any field of the business card is empty and if the number of cards requested is empty or isn’t a valid number. After validation, this event still applies Type Conversion on the number of cards requested, hide/show different buttons and show a message to the user.
*If your browser doesn't support Color Picker you can input the color name or code directly.
